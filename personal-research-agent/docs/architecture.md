1.1 系统定位

  Personal Research Agent 是一个基于大语言模型的智能研究辅助系统，
  能够接收用户的研究目标，自动完成资料检索、信息提取和结构化报告生成。

  [2] 可扩展     工具模块化设计，易于添加新的能力
  [3] 安全可控   输入验证、预算控制、敏感信息过滤
  [4] 成本优化   词元跟踪和预算管理

3.1 主入口模块 (main.py)

  【职责】
    - 解析命令行参数
    - 提供 CLI 和交互模式
    - 协调各模块初始化
    - 输出格式化和结果保存

  【主要接口】
    python main.py --check-model        # 模型连通性测试
    python main.py --goal "研究目标"    # 执行研究任务
    python main.py --interactive        # 交互模式


3.2 智能体核心 (agent.py)

  【职责】
    - 接收用户目标
    - 生成研究计划
    - 执行各研究步骤
    - 管理智能体状态
    - 生成研究报告

  【核心类】
    class ResearchAgent:
        - run(goal: str) -> Dict        # 主入口
        - _plan_research(goal)          # 计划生成
        - _execute_step(step)           # 步骤执行
        - _generate_report()            # 报告生成

  【状态管理】
    class AgentState:
        - goal: str                     # 研究目标
        - plan: List[str]               # 研究计划
        - current_step: int             # 当前步骤
        - step_results: List[Dict]      # 执行结果
        - token_usage: TokenUsage       # 词元消耗
        - is_complete: bool             # 完成状态
        - is_failed: bool               # 失败状态


3.3 模型客户端 (model_client.py)

  【职责】
    - 封装大模型API调用
    - 词元计数和成本估算
    - 连通性测试
    - 统一错误处理

  【核心类】
    class ModelClient:
        - chat(messages) -> Response    # 发送请求
        - count_tokens(text) -> int     # 词元计数
        - check_connectivity() -> Dict  # 连通性测试

  【数据类】
    class TokenUsage:
        - prompt_tokens: int            # 输入词元
        - completion_tokens: int        # 输出词元
        - total_tokens: int             # 总词元
        - cost: float                   # 估算成本


3.4 工具包 (tools.py)

  【MCP风格工具契约】

  工具1: web_search
    - 用途: 搜索网络资料
    - 输入: {query: string, top_k: integer(1-10)}
    - 输出: {items: array, total_found: integer}
    - 失败: validation_error, network_error, security_blocked

  工具2: knowledge_base_search
    - 用途: 检索本地知识库
    - 输入: {query: string, top_k: integer(1-5), category: string(可选)}
    - 输出: {items: array, total_found: integer}
    - 失败: validation_error, knowledge_base_missing, no_documents

  工具3: research_summarizer
    - 用途: 生成研究摘要
    - 输入: {content: string, max_length: integer(50-500), format: string}
    - 输出: {summary: string, key_points: array, word_count: integer}
    - 失败: validation_error, length_exceeded, security_blocked

  【工具注册表】
    class ToolRegistry:
        - register(name, instance)      # 注册工具
        - get(name) -> Tool             # 获取工具
        - list_tools() -> List          # 列出所有工具


3.5 安全护栏 (guardrails.py)

  【职责】
    - 输入验证（空输入、长度限制）
    - 敏感词过滤
    - 文件路径限制
    - 日志脱敏
    - 预算检查

  【核心方法】
    class SecurityGuardrail:
        - validate_input(text) -> Dict       # 输入验证
        - validate_file_path(path) -> Dict   # 路径验证
        - validate_url(url) -> Dict          # URL验证
        - sanitize_output(text) -> str       # 输出脱敏
        - check_budget(tokens) -> Dict       # 预算检查

9.1 添加新工具

  步骤1: 在 tools.py 中实现新工具类
  ┌─────────────────────────────────────────────────────────────┐
  │  class NewTool:                                            │
  │      def __init__(self):                                   │
  │          pass                                              │
  │                                                             │
  │      def execute(self, **params) -> Dict:                  │
  │          # 实现工具逻辑                                     │
  │          pass                                              │
  └─────────────────────────────────────────────────────────────┘

  步骤2: 注册到工具注册表
  ┌─────────────────────────────────────────────────────────────┐
  │  tool_registry.register("new_tool", NewTool())             │
  └─────────────────────────────────────────────────────────────┘

  步骤3: 在 agent.py 中添加路由
  ┌─────────────────────────────────────────────────────────────┐
  │  if "新工具关键词" in step:                                 │
  │      return self._call_new_tool(step)                      │
  └─────────────────────────────────────────────────────────────┘
